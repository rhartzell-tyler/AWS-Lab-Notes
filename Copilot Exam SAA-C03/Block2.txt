Question 14  
A company runs a global e-commerce platform using Amazon Aurora MySQL. During large promotional events, the read workload increases significantly, causing high read latency. The company wants to improve read scalability with minimal operational overhead and without modifying application code. What should a solutions architect recommend?

A. Add Aurora Replicas in multiple AWS Regions and enable cross-Region read forwarding.
B. Increase the instance size of the primary writer node.
C. Add additional Aurora Replicas in the same Region and use the cluster endpoint.
D. Convert the cluster to Aurora Serverless v2.

❌ Question 14 — Correct: C
Your answer: D

Why C is correct
When Aurora MySQL has read pressure:

Adding Aurora Replicas in the same Region

And using the cluster endpoint

…automatically load-balances reads with zero application changes.

Why D is wrong
Aurora Serverless v2 is great, but:

It does not solve read scaling as effectively as replicas

It requires capacity autoscaling tuning

It’s not the “minimal operational overhead” choice for read-heavy workloads

Question 17  
A company uses Amazon RDS for PostgreSQL. The database receives heavy write traffic during business hours. The company wants to reduce write latency and improve throughput without changing the application code. What should a solutions architect recommend?

A. Enable Multi-AZ and add read replicas.
B. Convert the database to Amazon Aurora PostgreSQL.
C. Increase the instance size and add more storage.
D. Use Amazon ElastiCache for Redis as a write-through cache.

❌ Question 17 — Correct: B
Your answer: A

Why B is correct
When RDS PostgreSQL has heavy write traffic:

Aurora PostgreSQL provides a distributed, log-structured storage layer

Much higher write throughput

Lower commit latency

No application changes required

This is a classic AWS exam pattern.

Why A is wrong
Multi-AZ + read replicas helps reads, not writes.

Question 18  
A company runs a serverless application using API Gateway, Lambda, and DynamoDB. The application occasionally experiences throttling on DynamoDB during traffic spikes. The company wants to eliminate throttling without significantly increasing cost. What should a solutions architect recommend?

A. Enable DynamoDB Auto Scaling and set a higher maximum capacity.
B. Convert the table to on-demand capacity mode.
C. Add a write-through cache using ElastiCache for Redis.
D. Increase the provisioned capacity manually.

❌ Question 18 — Correct: B
Your answer: A

Why B is correct
DynamoDB on-demand mode is the exam’s preferred answer when:

Traffic is spiky

You want to eliminate throttling

You want to avoid overprovisioning

You want minimal operational overhead

Why A is wrong
Auto Scaling still lags behind sudden spikes — throttling can still occur.

Question 20  
A company wants to deploy a new microservices application using Amazon ECS on Fargate. The application requires service-to-service communication with low latency and built-in observability. The company wants to avoid managing service discovery infrastructure. What should a solutions architect recommend?

A. Use AWS App Mesh with Envoy sidecars.
B. Use Amazon Route 53 private hosted zones.
C. Use AWS Cloud Map with API-based discovery.
D. Use an Application Load Balancer for each service.

❌ Question 20 — Correct: A
Your answer: B

Why A is correct
For microservices needing:
low-latency service-to-service communication
mesh-level observability
no custom service discovery infra
AWS App Mesh is the right answer.

Why B is wrong
Route 53 private hosted zones provide DNS, not:
retries
circuit breaking
telemetry

Question 26  
A company uses Amazon RDS for MySQL. The database is experiencing performance issues due to slow queries. The company wants to identify the root cause with minimal performance impact. What should a solutions architect recommend?

A. Enable RDS Performance Insights.
B. Enable general logging and slow query logging.
C. Increase the instance size.
D. Create a read replica and analyze queries there.

❌ Question 26 — Correct: A
Your answer: D

Why A is correct
RDS Performance Insights:
identifies slow SQL
shows wait events
has minimal overhead
is the exam’s preferred diagnostic tool

Why D is wrong
A read replica still runs the same slow queries — it doesn’t diagnose them.
