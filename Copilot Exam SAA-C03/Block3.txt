Question 32  
A company runs a multi-tenant SaaS application using Amazon Cognito for authentication. Some tenants require integration with their corporate identity providers using SAML. The company wants to support this without creating separate user pools. What should a solutions architect recommend?

A. Create a new user pool for each tenant
B. Use Cognito identity pools instead of user pools
C. Configure SAML identity providers within the same user pool
D. Use AWS SSO for all tenants

❌ Question 32 — Correct: C
Your answer: B

Why C is correct
Cognito user pools support:

multiple SAML IdPs

multiple OIDC IdPs

all inside the same user pool

This is the multi-tenant SaaS pattern.

Why B is wrong
Identity pools ≠ authentication.
They federate after authentication.

Question 34  
A company runs a high-traffic API on Amazon API Gateway and AWS Lambda. The API must integrate with a legacy SOAP service hosted on-premises. The company wants to minimize latency and avoid exposing the SOAP service directly to the internet. What should a solutions architect recommend?

A. Use a VPC link with a Network Load Balancer pointing to the on-premises service
B. Use an API Gateway HTTP integration with the SOAP endpoint
C. Use AWS Direct Connect and call the SOAP service directly from Lambda
D. Use an SQS queue between API Gateway and the SOAP service

❌ Question 34 — Correct: A
Your answer: C

Why A is correct
To integrate API Gateway with an on-prem SOAP service:

Use VPC Link

Connect to an NLB

NLB forwards to on-prem via Direct Connect or VPN

This keeps the SOAP service private and minimizes latency.

Why C is wrong
Calling the SOAP service directly from Lambda:

exposes the service unless you build a whole VPC + DX/VPN path

doesn’t integrate with API Gateway cleanly

doesn’t solve the “don’t expose SOAP service” requirement

Question 41  
A company wants to deploy a multi-Region application using Amazon Aurora Global Database. The application requires low-latency reads in each Region and the ability to promote a secondary Region quickly during a disaster. What should a solutions architect recommend?

A. Use the global reader endpoint for all reads
B. Use the cluster endpoint in each Region
C. Use the reader endpoint in each secondary Region
D. Use the writer endpoint in each Region

❌ Question 41 — Correct: C
Your answer: A

Why C is correct
In Aurora Global Database:

Secondary Regions use the reader endpoint

Low-latency reads

Automatic failover promotion path

Why A is wrong
The global reader endpoint is cross-Region — not optimal for local reads.

Question 42  
A company wants to store application logs for 5 years. Logs are frequently accessed for the first 30 days and rarely accessed afterward. The company wants to minimize cost. What should a solutions architect recommend?

A. Store logs in S3 Standard and transition to S3 Glacier Deep Archive after 30 days
B. Store logs in S3 Intelligent-Tiering
C. Store logs in S3 Standard-IA and transition to S3 Glacier after 30 days
D. Store logs in S3 One Zone-IA and transition to S3 Glacier Deep Archive after 30 days

❌ Question 42 — Correct: A
Your answer: D

Why A is correct
For logs:

S3 Standard for 30 days

Transition to Glacier Deep Archive

This is the cheapest compliant pattern.

Why D is wrong
One Zone-IA is not appropriate for logs requiring durability.
