# ⭐ AWS SAA – Top 20 Services You MUST Know Cold  
*(These drive 80–90% of the exam questions)*

## 1. **Amazon EC2**
Instance types, pricing models, placement groups, ENIs, user data, metadata, scaling patterns.

## 2. **Elastic Load Balancing (ALB / NLB)**
ALB = HTTP/HTTPS, routing, host/path rules.  
NLB = TCP/UDP, extreme performance, static IPs.

## 3. **Auto Scaling Groups (ASG)**
Scaling policies, lifecycle hooks, health checks, warm pools, termination policies.

## 4. **Amazon S3**
Storage classes, lifecycle rules, versioning, replication, encryption, presigned URLs.

## 5. **Amazon RDS**
Multi‑AZ, read replicas, backups, failover behavior, parameter groups, storage types.

## 6. **Amazon Aurora**
Cluster architecture, replicas, failover, serverless v2, global database.

## 7. **Amazon DynamoDB**
RCU/WCU, autoscaling, DAX, streams, TTL, GSIs vs LSIs, transactions.

## 8. **Amazon VPC**
Subnets, route tables, NAT, IGW, NACLs vs Security Groups, VPC Flow Logs.

## 9. **VPC Peering & Transit Gateway**
Private connectivity patterns, non‑overlapping CIDRs, routing requirements.

## 10. **AWS PrivateLink**
Interface endpoints for private access to AWS services or your own services.

## 11. **Amazon Route 53**
Routing policies, health checks, failover, alias vs non‑alias, private hosted zones.

## 12. **AWS Lambda**
Concurrency, cold starts, IAM permissions, event sources, DLQs, versions & aliases.

## 13. **Amazon API Gateway**
REST vs HTTP APIs, throttling, caching, auth (IAM, Cognito, Lambda authorizers).

## 14. **Amazon SQS**
Standard vs FIFO, visibility timeout, DLQs, scaling patterns.

## 15. **Amazon SNS**
Fan‑out, message filtering, delivery retries, mobile push.

## 16. **Amazon Kinesis**
Streams vs Firehose vs Analytics, shard scaling, ordering, replay.

## 17. **AWS Step Functions**
Standard vs Express workflows, service integrations, error handling, retries.

## 18. **AWS IAM**
Policies, roles, STS, conditions, permission boundaries, Organizations + SCPs.

## 19. **AWS KMS**
CMKs, envelope encryption, automatic key rotation, cross‑account access.

## 20. **CloudWatch & CloudTrail**
Metrics, logs, alarms, dashboards, events, API auditing, retention.

---

# ⭐ Why these 20 matter
These services form the backbone of:
- high availability  
- fault tolerance  
- cost optimization  
- security  
- serverless integration  
- networking  
- data durability  
- scaling patterns  

If you know these cold, you can answer **almost every scenario‑based question** on the exam.
